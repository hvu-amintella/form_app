# Architecture

This project has two related workflows:

- a fixed Saclay cleaning inspection app
- a generic form generator that can build a temporary app experience from another structured `.docx` template or fillable `.pdf`

All paths below are relative to the repository root.

## Runtime Shape

The app is a Vite + React single-page application.

```text
index.html
src/main.jsx
src/App.jsx
```

`src/main.jsx` mounts the React app. `src/App.jsx` owns the main tab layout and coordinates the feature modules.

## Main Modules

```text
src/App.jsx
src/constants.js
src/archiveStorage.js
src/statistics.js
src/genericDocx.js
src/genericPdf.js
src/App.css
src/index.css
```

### `src/App.jsx`

Primary React module. It currently contains:

- fixed Saclay inspection UI
- photo upload and preview UI
- DOCX export/import orchestration for the Saclay template
- archive/history UI
- statistics UI and export trigger
- generic generator tab for `.docx` and `.pdf`

This file is still large. Future refactors should move the tab panels and fixed Saclay DOCX helpers into smaller modules.

### `src/constants.js`

Shared constants for:

- fixed Saclay form sections and labels
- storage keys
- template and logo asset paths
- Word XML namespaces
- photo sizing and file type settings

### `src/archiveStorage.js`

Small IndexedDB wrapper for archive file persistence.

Archive metadata is stored in `localStorage` by `src/App.jsx`; generated binary files are stored in IndexedDB.

### `src/statistics.js`

Builds archive statistics snapshots for both workflow families:

- counts imported/app-created archives
- keeps Saclay-specific section and building statistics for the fixed inspection flow
- groups generated-form records by source template
- calculates per-field completion and option percentages for generated forms
- groups similar Saclay remarks

### `src/genericDocx.js`

Generic DOCX generator engine.

Input:

- a `.docx` file selected in the browser

Processing:

- opens the DOCX as a ZIP with `jszip`
- reads `word/document.xml`
- detects table-based empty cells as text fields
- detects Word checkbox controls
- builds a schema grouped into sections

Output:

- a filled `.docx` generated by writing values back into the original document XML

Best suited for structured Word forms that use tables and Word checkbox controls.

### `src/genericPdf.js`

Generic PDF generator engine.

Input:

- a fillable or flat text-based `.pdf` file selected in the browser

Processing:

- opens the PDF with `pdf-lib`
- reads real PDF form fields
- maps text fields, checkboxes, radio groups, dropdowns, and option lists into the shared generated-form UI
- if no real form field exists, inspects PDF text streams and proposes text fields from likely labels
- for flat PDFs, appends a response page to the original PDF instead of writing into unknown visual coordinates

Output:

- a filled `.pdf`

Scanned image-only PDFs are not supported yet. They need OCR and visual layout detection before fields can be inferred reliably.

## Asset Layout


The files in `public/` are served directly by Vite. The Saclay Word template is imported from `src/` so Vite can bundle it and return a URL at runtime.

## Data Flow

### Fixed Saclay Inspection

```text
User fills Inspection tab
        |
        v
React state in src/App.jsx
        |
        +--> localStorage draft
        |
        +--> DOCX export using bundled Saclay template
        |
        +--> archive metadata in localStorage
        |
        +--> archive file in IndexedDB
```

### Generic DOCX Generator

```text
User imports .docx
        |
        v
src/genericDocx.js infers schema
        |
        v
Generated form UI in src/App.jsx
        |
        v
src/genericDocx.js writes values into original DOCX XML
        |
        v
Browser downloads filled .docx
        |
        v
Generated form schema, values, and output file are archived locally
```

### Generic PDF Generator

```text
User imports fillable .pdf
        |
        v
src/genericPdf.js reads PDF form fields
        |
        v
Generated form UI in src/App.jsx
        |
        v
src/genericPdf.js fills PDF fields
        |
        v
Browser downloads filled .pdf
        |
        v
Generated form schema, values, and output file are archived locally
```

## Storage

The app currently stores data only in the browser:

- fixed inspection draft: `localStorage`
- archive metadata: `localStorage`
- archive generated-form metadata: `localStorage`
- archive output files: IndexedDB

The generic generator archives the inferred schema and entered values after export. Uploaded source templates are not persisted separately; the generated output file is stored.

## Statistics Scope

The `Statistiques` tab now has two layers:

- Saclay statistics: fixed sections, buildings, quality, improvement points, and remarks
- generated-form statistics: one group per imported template, with field completion and option/value distribution

Generated-form statistics do not assume Saclay fields such as buildings or quality. They are calculated from each archived schema and its submitted values.
